## 2. Duplicate logic

### D1. `HumanBytes` (internal/ui/progress.go:75) vs `formatSize` (internal/tui/browse/update.go:538)

Same intent (bytes → short human string), different format.
- `HumanBytes(1536)` → `"1.5 KB"`.
- `formatSize(1536)` → `"1.5 KiB"` (and returns `"-"` for zero/negative).

Both roll their own rather than calling `github.com/dustin/go-humanize` (already in the dep graph transitively via charmbracelet/bubbles). Recommend: pick one, move to `internal/ui/progress.go`, delete the other. `formatSize`'s zero-handling is nicer for TUI rows.

### D2. `truncate` (internal/cmd/search.go:201) vs `truncStr` (internal/tui/browse/update.go:514)

Both trim to n chars with ellipsis. `truncate` uses byte slicing (breaks on multibyte runes); `truncStr` uses runes correctly and handles `n==1` as a single ellipsis. Keep `truncStr`, move to `internal/ui`, delete `truncate`. (polish for the cmd side)

### D3. `brewPrefix()` (internal/backend/brew/brew.go:63) vs `brewCellar()` (internal/cmd/size.go:117)

Both probe `/opt/homebrew` then `/usr/local` and return the prevailing path. `brewCellar` appends `/Cellar`; `brewPrefix` doesn't. The comment in `size.go` explicitly notes it's duplicating because `brewPrefix` is unexported — yet `brew.Prefix()` (duplicates.go:15) **is** the exported wrapper. Net: `brewCellar` should call `brew.Prefix() + "/Cellar"`. One-line fix. (polish)

### D4. MCP resource handlers share structure

`mcp-server/src/resources/outdated.ts`, `mcp-server/src/resources/verify.ts`, and the static half of `installed.ts` are structurally identical (`registerResource(name, uri, metadata, async uri => runYumJSON + JSON.stringify)`). A tiny helper `registerReadonlyJsonResource(server, runner, { name, uri, args, title, description })` would collapse ~70 lines into ~25. Low urgency — these are each 30 lines.

### D5. `parseUninstalled` / `parseFreedSize` are siblings, not duplicates

Both scrape brew stdout but for different tokens. Note for reference only, no action.

### D6. Near-duplicate cobra command wiring in `mutations.go`

`newRemoveCmd`, `newReinstallCmd`, `newUpgradeCmd` all build the same `RunE` shape: `boot + runMutate(..., doer)`. `runMutate` already absorbs most of the shared work. The residual duplication (3×10 lines) is acceptable and readable.

## 3. Integration gaps

### `--json` coverage
Confirmed JSON path for: `list`, `search`, `info`, `outdated`, `history`, `history info`, `log`, `duplicates`, `verify`, `services list`, `tree`, `why`, `size`, `provides`, `repolist`, `doctor`, `version`. **Not** JSON-aware (returns only human text): `install`, `remove`, `reinstall`, `upgrade`, `autoremove`, `clean`, `pin`, `unpin`, `rollback`, `sync`, `manifest`. For mutations this is probably fine — the journal is the canonical record — but scripted callers who pass `--json install ripgrep` get unexpected human output. Document or gate.

### `--dry-run` coverage
Handled in: `install`, `mutations` (via `runMutate`), `autoremove`, `rollback`, `clean`, `pin`/`unpin`, `duplicates`, `services`, `manifest apply`. **Not** handled: `sync` (it will actually upgrade + autoremove + cleanup — serious gap given the command's power). **Fix: add early Flags.DryRun branch to `newSyncCmd`.**

### `--no-color` end-to-end
`boot()` reads both `Flags.NoColor` and `NO_COLOR` env and sets `theme.NoColor` before `theme.Load()`. theme tests confirm render is plain. Good. (polish may further refactor)

### Journal correctness
Commands that **do** journal: `install`, `remove`, `reinstall`, `upgrade`, `autoremove`, `duplicates prune`, `services start/stop/restart`, `browse` mutations. Commands that **do NOT** journal mutations:
- `cmd/sync.go` — upgrades, autoremoves, and cleans up without writing any `transactions` rows. Sync becomes invisible in `yum history`. **Bug; fix.**
- `cmd/manifest.go` (apply path) — installs formulae and casks and sets pins without journaling. **Bug; fix.**
- `cmd/misc.go` `pin`/`unpin` — mutates formula pin state without journaling. Rollback can't reverse a `yum pin foo` because it was never recorded. **Bug; fix.**
- `cmd/rollback.go` — the rollback itself is not journaled as a new transaction. That means you can't "rollback the rollback". Minor — Homebrew's own history has the same quirk.

### Info cache invalidation
`internal/backend/brew/cache.go:invalidateCache()` is called from `Install`, `Remove` (native and brew paths), `Reinstall`, `Upgrade`, `InstallNative`, `UninstallNative`. **Missing** from:
- `Autoremove` (`brew.go:579`) — can remove packages whose info is cached as "installed". **Bug.**
- `Pin`/`unpin` (`brew.go:441`) — pinned state is in `info`, so pin/unpin leaves a stale cache until 5 min TTL. Minor.
- `Cleanup` — prunes old versions, can invalidate `Installed` field visibility. Minor.

### Context cancellation
`runner.Real.Run` / `Stream` use `exec.CommandContext`, so ctrl+c via `cmd/ctx.go` SIGINT → `context.Cancel` → child killed. `internal/tui/browse/browse.go` threads a cancellable `context.WithCancel(context.Background())` into the model. Good.

### Help text
`yum --help` lists all 29 commands correctly. `yum help <cmd>` works for each spot-checked (install, remove, verify, services, duplicates, log). No broken `cobra` subcommands.

- **Finding MCP-3 (mild):** `jsonResult` in `result.ts` wraps non-object values as `{ value: X }`, but no tool currently uses non-object values, so the branch is untested. Low-impact observation.

## 10. Priority-ranked recommendations

```
1. [HIGH] internal/cmd/sync.go does not journal upgrade/autoremove/cleanup — effort: S
   Why: "yum sync" mutations don't appear in "yum history" and can't be rolled back. Silent regression of the journal feature.
   Fix: Wrap each step in Journal.Begin/End with action="upgraded"/"removed" and parse parseUninstalled / read installed list to record package rows.

2. [HIGH] internal/cmd/manifest.go apply does not journal the installs it performs — effort: S
   Why: Declarative-apply is the exact case where rollback is most useful; right now `yum manifest apply prod.toml` is invisible to history.
   Fix: Call Journal.Begin/End around the Install / Pin loop; record one tx per manifest with all packages.

3. [HIGH] internal/verify/verify.go:250 uses string comparison to detect "older than linked" Cellar versions — effort: S
   Why: Misreports semver 1.10 vs 1.9; false positives/negatives on verify reports.
   Fix: Swap to Masterminds/semver (already in deps via duplicates.go) or the Diff helper in internal/semver.

4. [HIGH] internal/cmd/sync.go has no --dry-run branch — effort: XS
   Why: `--dry-run sync` actually upgrades everything. Users reasonably expect dry-run to be safe on every command.
   Fix: Early `if Flags.DryRun { print; return nil }` at the top of RunE.

5. [MEDIUM] internal/backend/brew/brew.go Autoremove + Pin + Cleanup don't invalidate info cache — effort: XS
   Why: Stale cache for up to 5 min after a mutation changes installed/pinned state.
   Fix: Call invalidateCache(names) in Autoremove (after parse), Pin (single name), and Cleanup (could just drop the whole cache dir).

6. [MEDIUM] mcp-server resources/list returns 291 entries on a typical Mac — effort: XS

7. [MEDIUM] README.md and CHANGELOG.md are missing log / verify / duplicates / services / browse / MCP server — effort: S
```
