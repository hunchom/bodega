# yum — Modern Package Manager for macOS

**Status:** Approved design, 2026-04-17

## Problem

macOS has `brew`. brew works but its UX is noisy, inconsistent across `formula` vs `cask` vs `tap`, has no real transaction history, no rollback, no dep-tree visualization, and a friendly-but-chatty output style. The user prefers the yum/dnf command vocabulary but wants something that beats modern dnf on UX, not just matches old yum.

Today the user has a zsh `yum()` function that dispatches to brew — a pure router with no state, no visual layer, no added value. We replace it with a real tool.

## Goals

- yum/dnf command vocabulary (install, remove, update, search, info, list, provides, deplist, history, clean, repolist, autoremove, reinstall, check-update)
- Visual quality ceiling of modern TUI tooling (Charm-tier), not shell scripting
- Features brew lacks: rollback, real history, reverse dep-tree, disk-size breakdown, manifest export/apply, semver-diff outdated view, security audit
- Single transparent surface over formula + cask — user never types `--cask`
- Readable as data when piped: `--json` on every read command; detect TTY and switch to plain output automatically
- Fast cold start (<40ms to help output)

## Non-goals

- Building packages from source, own package format
- GUI, web UI, multi-machine sync, daemon
- Generic plugin system — backends are compiled in
- Replacing brew's internals (still the underlying installer)
- Supporting Linux (macOS only — use real yum/dnf there)

## Stack

- **Language:** Go 1.26
- **CLI:** [spf13/cobra](https://github.com/spf13/cobra) for command tree + autogenerated completions
- **Rendering:** [charmbracelet/lipgloss](https://github.com/charmbracelet/lipgloss) for styling; [charmbracelet/bubbles](https://github.com/charmbracelet/bubbles) + [bubbletea](https://github.com/charmbracelet/bubbletea) for interactive picker and progress
- **Prompts:** [charmbracelet/huh](https://github.com/charmbracelet/huh) for confirm/select when interactive
- **Storage:** SQLite via [modernc.org/sqlite](https://pkg.go.dev/modernc.org/sqlite) (pure-Go, no CGO)
- **Config:** TOML via [pelletier/go-toml/v2](https://github.com/pelletier/go-toml)
- **Semver:** [Masterminds/semver/v3](https://github.com/Masterminds/semver)

## Command Surface

### yum parity

| command | behavior |
|---|---|
| `install <pkg>...` | Install; auto-detect formula vs cask; resolves + shows deps before confirm |
| `reinstall <pkg>...` | Reinstall |
| `remove <pkg>...` / `erase` | Uninstall; warns if reverse deps exist |
| `autoremove` | Remove orphaned deps (brew autoremove) |
| `update [pkg]...` | Update taps + upgrade packages (alias: `upgrade`) |
| `check-update` | List outdated with semver diff coloring |
| `search <term>` | Fuzzy search across formula + cask + tap; interactive picker on ambiguous |
| `info <pkg>` | Rich panel: version, size, tap, homepage, description, deps, rdeps, outdated, pinned state |
| `list [selector]` | selectors: `installed` (default), `available`, `updates`, `leaves`, `pinned`, `casks` |
| `provides <cmd>` | Which formula installs this binary |
| `deplist <pkg>` | Dependency tree (pretty) |
| `clean [all]` | brew cleanup; `all` = `--prune=all` |
| `repolist` | Active taps with status |
| `history` | Actual transaction history from journal |

### Beyond yum

| command | behavior |
|---|---|
| `tree <pkg>` | Forward dep tree, like deplist but explicit |
| `why <pkg>` | Reverse dep tree: what pulled this in |
| `size [pkg]` | Disk usage: total + per-package with inline bars |
| `outdated` | Same as check-update (dnf vocabulary); semver diff colored |
| `pin <pkg>` / `unpin` | Lock/unlock versions |
| `sync` | update + upgrade + autoremove + cleanup, one transaction, one progress |
| `rollback [id]` | Undo last transaction (or specific ID) using journal + brew cache |
| `history info <id>` | Detail of a transaction: packages, duration, exit, versions |
| `history undo <id>` | Alias for rollback |
| `doctor` | brew doctor + PATH conflicts + broken symlinks + orphaned files, each with fix hint |
| `manifest export [file]` | Snapshot: taps + installed formulae+casks + pins, TOML |
| `manifest apply <file>` | Reconcile system to manifest (add missing, optional --prune) |
| `audit` | Cross-ref installed packages against brew vuln feed |
| `completions <shell>` | zsh / bash / fish completion script |
| `top` | Live view of brew processes if any (optional, v2) |

### Global flags

- `--json` — machine-readable output on all read commands
- `--yes` / `-y` — skip confirmation prompts
- `--no-color` — disable ANSI (also respects `NO_COLOR=1` env)
- `--debug` — verbose logs to stderr, full stack on panic
- `--dry-run` — show what would happen, take no action
- `--config <path>` — override config path

## Visual Language

Hard rules to stay un-cringe:

1. **One accent color** — muted amber `#D79921` (Gruvbox material). No rainbows. Error = restrained red `#CC241D`. Success = dim green `#689D6A`. Muted text = `#928374`.
2. **No emojis anywhere in output.** Ever. Glyphs OK (•, →, ✓, ✗) in monospace weight.
3. **Thin single-weight box drawing** for tables and panels (`─ │ ┌ ┐ └ ┘`). No doubled `═`, no rounded `╭`.
4. **Terse column headers** — `name`, `ver`, `size`, `tap`. Lowercase. No units in cells (units in header: `size (MB)`).
5. **Right-align numeric columns**, tabular figures.
6. **No "Welcome to…" banners.** No ASCII logo. No startup messages.
7. **Help output** is columnar, not prose paragraphs. Command list grouped by section.
8. **Errors are one line** unless `--debug`. Format: `yum: <verb>: <what>: <why>`.
9. **Progress bar is one line**, unicode block glyph `█▊▋▌▍▎▏`. Collapses to a ✓ line on complete.
10. **Confirm prompts** use huh's minimal form, no color flourishes — plain prompt + y/N.

## Core Data Model

```go
// internal/backend/types.go
type Package struct {
    Name         string
    Version      string
    Latest       string    // latest available; empty if current
    Source       Source    // BrewFormula, BrewCask, Mas
    Tap          string    // "homebrew/core" or empty
    Desc         string
    Homepage     string
    License      string
    Size         int64     // bytes
    InstalledOn  time.Time
    Pinned       bool
    Deps         []string
    BuildDeps    []string
    OptionalDeps []string
}

type Source int
const (
    BrewFormula Source = iota
    BrewCask
    Mas
)

type Backend interface {
    Name() string
    Search(ctx context.Context, q string) ([]Package, error)
    Info(ctx context.Context, name string) (*Package, error)
    List(ctx context.Context, filter ListFilter) ([]Package, error)
    Install(ctx context.Context, names []string, w ProgressWriter) error
    Remove(ctx context.Context, names []string, w ProgressWriter) error
    Upgrade(ctx context.Context, names []string, w ProgressWriter) error
    Outdated(ctx context.Context) ([]Package, error)
    Deps(ctx context.Context, name string) (*DepTree, error)
    ReverseDeps(ctx context.Context, name string) ([]string, error)
    Pin(ctx context.Context, name string, pin bool) error
    Cleanup(ctx context.Context, deep bool) error
}
```

## Journal Schema

```sql
CREATE TABLE transactions (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    started_at   INTEGER NOT NULL,      -- unix seconds
    ended_at     INTEGER,
    verb         TEXT NOT NULL,          -- install | remove | upgrade | reinstall | pin | sync | rollback
    exit_code    INTEGER,
    cmdline      TEXT NOT NULL,
    yum_version  TEXT NOT NULL,
    brew_version TEXT NOT NULL
);

CREATE TABLE transaction_packages (
    transaction_id INTEGER NOT NULL REFERENCES transactions(id) ON DELETE CASCADE,
    name           TEXT NOT NULL,
    from_version   TEXT,                -- nullable for install
    to_version     TEXT,                -- nullable for remove
    source         TEXT NOT NULL,        -- formula|cask|mas
    action         TEXT NOT NULL         -- installed|removed|upgraded|reinstalled|pinned|unpinned
);

CREATE INDEX idx_tx_started ON transactions(started_at);
CREATE INDEX idx_txp_name ON transaction_packages(name);
```

## Config Schema (TOML)

```toml
# ~/.config/yum/config.toml

[ui]
theme = "amber"              # amber | teal | mono
confirm_destructive = true   # require confirm on remove/rollback
compact_tables = false       # thinner row padding

[defaults]
auto_cleanup = false         # after every upgrade
parallel = true              # use brew's parallel downloads

[aliases]
i = "install"
rm = "remove"
up = "upgrade"
```

## Architecture

- **cmd/yum/main.go** — entrypoint; wires cobra, loads config, opens journal, dispatches
- **internal/cmd/** — one file per command (install.go, remove.go, …) — the only layer that touches cobra
- **internal/backend/** — package-manager adapters, pure Go interface implementations
- **internal/journal/** — SQLite wrapper, transaction recording, rollback executor
- **internal/ui/** — rendering layer; every command imports from here; no command prints directly
- **internal/config/** — TOML loader with env override support
- **internal/semver/** — thin wrapper over Masterminds for diff coloring

All IO goes through `ui.Writer` (wraps `io.Writer`, knows TTY, handles `--json` + `--no-color`). Commands never call `fmt.Println`.

## Testing Strategy

- **Unit tests** for rendering components (table, tree, panel) using golden files — run the renderer, compare to `testdata/*.txt`
- **Unit tests** for semver diff coloring (pure function)
- **Unit tests** for journal (in-memory SQLite)
- **Integration tests** for brew backend behind a `testing.Short()` gate — run only when `YUM_INTEGRATION=1` — actually calls `brew list` against the real install
- **Golden-file CLI tests** using `--json` output + recorded `brew` fixtures
- No mocks of stdlib, no mocks of `os/exec` — use a `CommandRunner` interface with a real impl and a fake for tests

## Install Flow

```bash
cd ~/src/yum
go build -trimpath -ldflags="-s -w" -o ~/.local/bin/yum ./cmd/yum
mkdir -p ~/.zsh/completions
~/.local/bin/yum completions zsh > ~/.zsh/completions/_yum
# one-time: remove yum() function from ~/.zshrc, add ~/.local/bin to PATH
```

A `make install` target does all of the above + a sed surgical strike on `.zshrc` guarded by a backup.

## Out of Scope (v1)

- `mas` (Mac App Store) backend — stub interface, return "not implemented"
- `top` — real-time process view
- `audit` — needs vuln data source selection; ship with brew's `audit --installed` wrapper only
- Rolling back reinstalls that brew hasn't cached tarballs for — surface "cannot roll back, tarball evicted" and exit
